# 📦 مراجع منتجات أمازون - نسخة محلية

تطبيق محلي يعمل على اللابتوب بتاعك، بيخزّن كل البيانات عندك بدون أي سحابة، والمشتري يقدر يدخل من أي جهاز!

---

## 🚀 التشغيل السريع

### متطلبات:
- **Node.js** مثبت على اللابتوب
  - حمّل من: https://nodejs.org/
  - اختر الإصدار LTS (الأكثر استقراراً)

---

## 📋 خطوات التشغيل

### 1️⃣ الإعداد الأول (مرة واحدة):

```bash
# 1. افتح Command Prompt أو Terminal
# 2. اذهب للمجلد اللي فيه الملفات:
cd path/to/your/project

# 3. ثبّت المكتبات:
npm install

# 4. شغّل السيرفر:
npm start
```

### 2️⃣ ستشوف الرسالة دي:

```
╔════════════════════════════════════════════════════════════╗
║       📦 مراجع منتجات أمازون - متشغل بنجاح!  ║
╚════════════════════════════════════════════════════════════╝

🌐 الرابط المحلي:     http://localhost:3000
🌍 الرابط للمشتري:   http://192.168.1.100:3000

📋 خطوات الاستخدام:
1. افتح http://192.168.1.100:3000 على أي جهاز
2. أضف المنتجات من تبويب "إضافة منتج"
3. المشتري بيفتح نفس الرابط ويراجع من "مراجعة"
```

---

## 💡 كيف يستخدم:

### أنت (الباحث):
1. افتح `http://localhost:3000` على اللابتوب بتاعك
2. اضغط "➕ إضافة منتج"
3. الصق الرابط من أمازون واكتب البيانات
4. اضغط "أضف المنتج"
5. البيانات تُحفظ تلقائياً في ملف على اللابتوب

### المشتري (بعيد عنك):
1. احصل على رابط الـ IP من الرسالة (مثل: http://192.168.1.100:3000)
2. افتح الرابط في أي متصفح
3. اذهب لتبويب "✓ مراجعة"
4. كل منتج، اضغط على حالته (متوفر/بقلة/غير متوفر)
5. أضف السعر والملاحظات (اختياري)
6. اضغط "✓ حفظ والمنتج اللي بعده"

## 🔧 معلومات تقنية:

### ملفات المشروع:
```
مشروع/
├── server.js           (الخادم الرئيسي)
├── package.json        (المكتبات)
├── public/
│   └── index.html      (الواجهة الأمامية)
└── products_data.json  (قاعدة البيانات - تُنشأ تلقائياً)
```

### رقم البورت (Port):
- الافتراضي: **3000**
- لو بتبي تغيره، بدّل `const PORT = 3000;` في `server.js`

### قاعدة البيانات:
- تُخزّن في ملف `products_data.json`
- جميع البيانات محلية على اللابتوب بتاعك

---

## ⚙️ حل مشاكل شهيرة:

### ❌ المشتري لا يقدر يدخل الرابط:

**السبب 1: الفايروول**
- اذهب إلى Windows Defender Firewall
- اضغط "Allow an app through firewall"
- أضف Node.js

**السبب 2: الـ IP خاطئ**
- تأكد من استخدام الـ IP الصحيح من الرسالة
- مثلاً: `192.168.1.100` وليس `localhost`

**السبب 3: Port مشغول**
- غيّر الرقم في `server.js` من `3000` لـ `3001`

### ❌ "npm: command not found"
- معناته Node.js ما تثبت صح
- ثبّت من: https://nodejs.org/

### ❌ البيانات اختفت
- تأكد إن ملف `products_data.json` موجود في نفس المجلد
- لو اختفى، السيرفر بينشئ واحد جديد تلقائياً

---

## 🎯 نصائح للاستخدام الأفضل:

1. **شغّل السيرفر قبل ما تبدأ** في إضافة منتجات
2. **بلّغ المشتري بالرابط** (اكتبها في ورقة أو رسالة واتس)
3. **ما تقفل السيرفر** إلا لما تنتهي من الشغل
4. **حمّل النتائج** في آخر اليوم كـ backup
